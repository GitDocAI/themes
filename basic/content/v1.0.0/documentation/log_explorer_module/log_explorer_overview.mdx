# Log Explorer Overview (v1.0.0)

The Log Explorer Module in UTMStack is a central, interactive tool designed to provide real-time, customizable, and comprehensive visibility into an organization's log data. It supports efficient monitoring, filtering, and management essential for robust cybersecurity operations. This module is indispensable for the proactive monitoring and effective management of network activities within a well-maintained cybersecurity infrastructure.

## Overview

The Log Explorer interface is engineered for seamless navigation and efficient functionality. Its centerpiece is the Data Grid, which offers an extensive view of log data based on defined datasources and queries. This design allows for logical grouping of queries based on user-defined filters and the ability to open multiple tabs for various queries. The module provides visually intuitive representations of logs, including interactive charts and graphs, to simplify data interpretation and understanding of trends.

Users can save search criteria for future use, making the tool adaptable and time-efficient.

## Key Functionalities

### 1. Source Selection

The source functionality is a crucial data filtering tool that allows users to narrow down searches to specific index patterns. For example, users can confine their search to logs originating from Office365. The module also offers the flexibility to create custom index patterns or sources, enabling personalized and targeted search experiences.

![Log Explorer Source Selection](https://mintcdn.com/utmstack/OEt3XKGepmb6ogAs/images/logexplorer/sources_logexplorer.png?fit=max&auto=format&n=OEt3XKGepmb6ogAs&q=85&s=3ec7324236b7ce67cc43f7a21fe648dc)

### 2. Filters

The filtering functionality is a core component of Log Explorer, offering comprehensive data parsing based on user requirements. Filters can be defined to track logs from a specific computer or to isolate logs pertaining to particular operations, such as login events.

![Log Explorer Filters](https://mintcdn.com/utmstack/OEt3XKGepmb6ogAs/images/logexplorer/filters_logexplorer.png?fit=max&auto=format&n=OEt3XKGepmb6ogAs&q=85&s=eacad42c7c2fd3e710f29ce1d546ce62)

**New Features in Filters:**
*   A **clear filter** button is available on the far left to instantly remove all active filters.
*   A **default time filter** is automatically applied, defaulting to the last 24 hours and dynamically updating with the selected time range.
*   The **maximum range for any selected time filter is 30 days** to ensure optimal system performance.
*   Users can add additional custom filters using the **Add Filter** button.
*   Filter creation is simplified by allowing users to add a filter based on a specific log field with a single click.

![Log Explorer Timestamp Filter](https://mintcdn.com/utmstack/OEt3XKGepmb6ogAs/images/logexplorer/timestamp_logexplorer.png?fit=max&auto=format&n=OEt3XKGepmb6ogAs&q=85&s=4be5ee28b5defd36176422feaf9ddbd)

### 3. Data Grid

The Data Grid is where most log information is viewed. Users have the choice of viewing data in a table or chart format, customized based on a selected field. In Table mode, users can define which columns to see, either through the "Select fields" option or by directly selecting a field in the log.

![Log Explorer Chart Mode](https://mintcdn.com/utmstack/c31KcEgLT8_kl3Nu/images/logexplorer/chartmode.png?fit=max&auto=format&n=c31KcEgLT8_kl3Nu&q=85&s=faacf2b4d7b58e602e5055645027dbb5)

### 4. The Log Section

This section offers a comprehensive view of all log information, including details like `agent_id`, `process name`, and `source IP`. Data can be viewed in either a table view or JSON mode, based on user preference.

![Log Explorer The Log](https://mintcdn.com/utmstack/OEt3XKGepmb6ogAs/images/logexplorer/the_log.png?fit=max&auto=format&n=OEt3XKGepmb6ogAs&q=85&s=40272e06fe093161dfbea92ecde31aec)

#### 4.1 Options for Log Fields

Each log field within the Log Explorer module provides two key options:
*   **Adding a filter**: Users can swiftly add a filter based on the field and value of a specific log, refining the data in the Data Grid.
*   **Creating a column**: Users can choose to create a new column in the Data Grid Table based on the field, providing immediate visibility of key data points.

![Log Explorer Field Options](https://mintcdn.com/utmstack/OEt3XKGepmb6ogAs/images/logexplorer/fields.png?fit=max&auto=format&n=OEt3XKGepmb6ogAs&q=85&s=d947c6966857560df8842678)

### 5. Personalize Fields

Users can add or remove columns in the Data Grid view based on the fields of the logs, allowing for tailored data presentation.

![Log Explorer Columns Customization](https://mintcdn.com/utmstack/c31KcEgLT8_kl3Nu/images/logexplorer/colums.png?fit=max&auto=format&n=c31KcEgLT8_kl3Nu&q=85&s=5dced91ba692b1f786100c8052ca6712)

### Saving and Exporting Information

Users can refresh their data, save the current query for future use, or export the current data to a CSV file for offline analysis. The ability to view and manage stored queries ensures that past investigations can be revisited with ease.

![Log Explorer Save Options](https://mintcdn.com/utmstack/OEt3XKGepmb6ogAs/images/logexplorer/options.png?fit=max&auto=format&n=OEt3XKGepmb6ogAs&q=85&s=bad497a5fb988007dcbcd080eace47d2)

## Office 365 Login Failure Analysis: Step-by-Step Guide Example

This example demonstrates how to utilize the Log Explorer module to create, configure, and save a query focused on Office 365 login failure events.

### Step 1. Source Selection

1.  In the Source field, type and select **`log-o365-*`** from the dropdown list. This sets the data source to all Office 365 logs.

    ![Office 365 Source Selection](https://mintcdn.com/utmstack/c31KcEgLT8_kl3Nu/images/logexplorer/0365_selection.png?fit=max&auto=format&n=c31KcEgLT8_kl3Nu&q=85&s=43b0dce1d54e8535a55506532bd6ba5e)

### Step 2. Filter Configuration

1.  Click on the ‘Add Filter’ button.
2.  In the Field dropdown, find and select **`logx.o365.Operation.keyword`**.
3.  In the Operator dropdown, select **`is`**.
4.  Type **`UserLoginFailed`** in the Value input box.
5.  Click **Add Filter** to apply this filter.

    After these steps, all logs about User login failures in Office 365 will be displayed.

    ![Office 365 Filter Selection](https://mintcdn.com/utmstack/OEt3XKGepmb6ogAs/images/logexplorer/filter_selection.png?fit=max&auto=format&n=OEt3XKGepmb6ogAs&q=85&s=32eae73fe5e4c6c1e7b8a3d3276244ce)

### Step 3. Column Selection

Users can personalize the columns displayed for an easier overview.

**Option A: Using 'Select fields'**
1.  Click on the ‘Select fields’ on the left side of the Data Grid.
2.  From the dropdown list, find and select **`Userid`**, **`Workload`**, and **`AuthenticationType`**.

    ![Office 365 Example Fields](https://mintcdn.com/utmstack/c31KcEgLT8_kl3Nu/images/logexplorer/examplefields.png?fit=max&auto=format&n=c31KcEgLT8_kl3Nu&q=85&s=74da55288b194cf7a7d0b1008b7ef13c)

**Option B: Directly from the Log list (Table mode)**
1.  In the Log list, switch to the Table mode view.
2.  From the field list, find and press the table icon next to the fields **`userid`**, **`workload`**, and **`authenticationType`**.

    ![Office 365 Example Columns](https://mintcdn.com/utmstack/c31KcEgLT8_kl3Nu/images/logexplorer/examplecolums.png?fit=max&auto=format&n=c31KcEgLT8_kl3Nu&q=85&s=29dca7a23e91b24c09f96d64fc07175e)

These selected fields will be added to your Data Grid view.

![Office 365 Example View](https://mintcdn.com/utmstack/OEt3XKGepmb6ogAs/images/logexplorer/exampleview.png?fit=max&auto=format&n=OEt3XKGepmb6ogAs&q=85&s=3f2131869e4fada2dbe67f7234bf8236)

## Summary

The Log Explorer Module is a dynamic and interactive tool within UTMStack, providing exhaustive and comprehensive visibility into an organization’s log data. It enables real-time filtering and nuanced analysis through intuitive features like flexible source selection, comprehensive filtering options (including new enhancements), a customizable Data Grid for various visualizations, and detailed log views. The ability to save search criteria and export data streamlines investigations, making log exploration efficient, adaptable, and tailored to specific user requirements.
